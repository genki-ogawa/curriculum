■ ネットワーク層
○IP
Internet Protocolの略。相手先のアドレス情報をもとにしてパケットを送信する機能。インターネット層のプロトコル。

○IPヘッダ
IPパケットの冒頭部にある、IPバージョン・送信元アドレス・宛先アドレスなどの情報が格納された部分

○ICMP
Internet Control Message Protocolの略。IP通信の疎通状態を調査したり、正常に通信できない場合に送信元ホストへエラー情報を返したりする、IP通信を補佐するためのプロトコル。

○ICMPメッセージ
「Queryメッセージ」と「Errorメッセージ」の2つ

○IPアドレス
IPで通信相手を識別するための番号

○オクテット
IPv4において8ビットずつ、カンマで区切られた部分

○ネットワーク部
ネットワークを識別するアドレス

○ホスト部
コンピューターや各機器を識別するアドレス

○クラス
IPアドレスの構成

○プライベートIPアドレス
インターネットのようなグローバルなものではなく、社内LANなど小規模なネットワークで使用されるIPアドレス

○グローバルIPアドレス
インターネットプロバイダに割り振られているIPアドレスや、インターネットに接続するパソコンなどの機器に割り振られるIPアドレスのこと

○ネットワークアドレス
ホスト部の2進数のビットがすべて０のアドレスのこと

○ブロードキャストアドレス
ホスト部の2進数のビットがすべて1のアドレスのこと

○ループバックアドレス
クラスAアドレスのうち、「127」で始まるアドレスのこと

○サブネット
IPネットワークを論理的に細分化したもの

○サブネットマスク
32ビットで構成され、IPアドレスとセットで使う。
IPアドレスのどこまでがネットワーク部かを示す値。

○プレフィックス表記
何ビット目までがネットワーク部かを簡易的にした表記のこと。IPアドレスの記載に続いて、「/（ネットワーク部のビット数）」で表記される。

○デフォルトゲートウェイ
同一ネットワーク内のノードから外部ネットワークに属するノードと通信する際に出入り口となるデバイスやポートのこと。
（実務ベースでは、適切な転送先が見つからない場合に使用するIPアドレスを指すことが多い）

○IPv6
各コンピュータに割り振られるIPアドレスを128ビットのデータとして表現するもの

○サブネットプレフィックス
IPv6のIPv4におけるネットワーク部に相当。基本的に前半64ビットがサブネットプレフィックス

○インターフェイスID
IPv6のIPv4におけるホスト部に相当。通常のIPv6ノードは、インタフェイスIDをMACアドレスから算出。
 
○ルータ
コンピュータネットワークにおいて、データを2つ以上の異なるネットワーク間に中継する通信機器
どのルートを通してデータを転送すべきかを判断するルート選択機能を持つ

○コンソール接続
ルータのコンソールポートとPCをコンソールケーブルで接続する方法

○VTY接続
物理的に離れたところからネットワークを通じて仮想的に接続する方法

○AUX接続
ルータのAUXポート(補助ポート)とモデムを接続し、電話回線経由で接続する方法

○ルーティングテーブル
ルータに記録される経路情報。（ルーティング処理を行う際に参照）

○ネクストホップ
次に転送する最も近いルータやインターフェイスのこと

○ARPテーブル
IPアドレスとMACアドレスの対照表

○ARP（Address Resolution Protocol）
MACアドレスをあて先のIPアドレスから求めるためのプロトコル。

○コアルータ
基幹ネットワークにおいてメインで用いられるルータ。
通信キャリアやISPネットワークの通信を中継するために利用されるルータで、大量のデータを転送する必要があるため、高速なデータ転送が可能

○センタールータ
企業で中心的に利用されるルータ。
拠点となる本社や、データセンター等の企業内で中心となる場所を接続するためのネットワークを構成

○エッジルータ
企業で利用されるルータ。
中心となる拠点を除く、営業所や支社等の拠点間を接続するネットワークを構成

○ブロードバンドルータ
家庭環境やSOHO環境等、インターネット回線に接続することを目的として利用されるルータ

○ルーティング
異なるネットワークにパケットを送信する際に最適な経路を決めること

○直接接続
ルータが直接接続しているネットワークのルートのこと

○スタティックルーティング
管理者が宛先ネットワークへの最適なルートを手動で設定したルートのこと

○ダイナミックルーティング
ルーティングプロトコルにより動的に追加されるルートのこと

○ディスタンスベクタ型
ホップ数（=宛先に到達するまでに経由するデバイス数）をメトリックとし、経路を選ぶ方式。
自身のルーティングテーブルに、隣接ルータから転送された情報をそのまま追加していく方法でルート情報のやりとりを行う

○リンクステート型
各ルータが全体の接続情報を見て、最適経路を選択する方式。
ルータ同士でLSA（Link State Advertisement）という自身のネットワーク情報を交換し、それを用いてネットワーク全体を把握している。
各ルータの負荷がかかるが、ルートの再構成にかかる時間は短く済む。大規模ネットワーク向き

○ハイブリッド型
ディスタンスベクタ型をベースに、リンクステート型を組み合わせた方式。
Ciscoが独自に拡張した方式であるため、1つでも非Cisco社製デバイスが混じっている環境では使用できない
 
■ トランスポート層
○コネクション型通信
データ転送を行う前に「正常に相手と接続できているか？」を確認したうえで通信を開始する通信方法。

○ACK（ACKnowledgement）
二者間の通信で、相手方に何らかの肯定的な応答を返す際に送られる信号やデータ、パケットなどのこと

○コネクションレス型通信（データグラム方式）
事前の接続確認を行わずいきなりデータ転送を始める通信方法のこと

○TCP (Transmission Control Protocol) 
信頼性の高い通信を担保するための通信プロトコル。

○スリーウェイハンドシェイク
TCPにおいて使用されるコネクションの確立のこと。（3回のやりとりによって確立）

○Telnet
ネットワークに接続された機器を遠隔操作するために使用するアプリケーション層プロトコル

○HTTP（Hypertext Transfer Protocol）
Webサイトを表示する際に利用されるプロトコル

○FTP（File Transfer Protocol）
コンピュータ間でファイルを転送する際に利用されるプロトコル

○SMTP（Simple Mail Transfer Protocol）
メールを送信するために利用する通信プロトコル

○TCPヘッダ
TCPでのデータの伝送単位を「TCPセグメント」（TCP segment）あるいは「TCPパケット」と呼び、その先頭にある制御情報が記された部分のこと

○UDP（User Datagram Protocol）
トランスポート層で動作する速度重視の通信を行う際に用いられるプロトコル

○RTP（Real Time Transport Protocol）
動画や音声といったマルチメディアコンテンツを途切れなく再生するための伝送プロトコル

○SNMP (Simple Network Management Protocol) 
ルーターやサーバ、スイッチなどのネットワーク上の機器を監視・制御するための通信プロトコル

○DHCP（Dynamic Host Configuration Protocol）
IPアドレス等のネットワークに接続するために必要な情報を自動的に割り当てるプロトコル
クライアントサーバ型のプロトコル

○TFTP（Trivial File Transfer Protocol)
特定のコンピュータ間でファイル転送する時に使用するアプリケーション層プロトコル

○RIP (Routing Information Protocol) 
ディスタンスベクタ型のルーティングプロトコル。
距離と方向を基準に最適ルートを決定する。

○UDPヘッダ
UDPデータグラムの先頭部分にある、制御情報などを一定の形式で記述した領域
